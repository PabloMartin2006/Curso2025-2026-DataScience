Evaluación de Requisitos del Dataset: FC 26 (FIFA 26) Player Data

Fuente: Kaggle

Requisitos Obligatorios (R1 - R4)

R1. Dominio Ciudad Inteligente (Smart City)

CUMPLIMIENTO: No, pero permitido.

Discusión: El dataset de datos de jugadores no pertenece al dominio tradicional de "Ciudad Inteligente". Se clasifica en "Deportes" y "Análisis de Datos". No obstante, el uso ha sido autorizado por el profesor, y enfocaremos el proyecto en el concepto de "inteligencia" aplicada a la toma de decisiones.

R2. Datos Disponibles como archivo CSV

CUMPLIMIENTO: Sí.

Discusión: El dataset se encuentra disponible para su descarga en formato "CSV" desde la plataforma Kaggle, satisfaciendo completamente este requisito de formato.

R3. Licencia Abierta (para publicación)

CUMPLIMIENTO: Sí.

Discusión: El dataset está publicado bajo la licencia "Creative Commons Attribution 4.0 International (CC BY 4.0)". Esta es una licencia abierta que permite a los usuarios compartir y adaptar el material, incluyendo su publicación como "Linked Data", siempre que se proporcione la atribución adecuada.

R4. Fácilmente Enlazable con Entidades del Mundo Real

CUMPLIMIENTO: Sí.

Discusión: El dataset es altamente enlazable. Las entidades clave son "Jugadores", "Clubes" y "Nacionalidades", las cuales pueden vincularse fácilmente a recursos externos como DBPedia o Wikidata. Por ejemplo, los nombres de los clubes y países sirven como identificadores directos para los enlaces.

Requisitos Opcionales (R5 - R6)

R5. Existencia de Documentación

GRADO DE SATISFACCIÓN: Alto.

Discusión: Kaggle proporciona una descripción detallada del Contexto y Contenido del dataset, incluyendo la fuente del "scraping" y una explicación de las columnas clave ("Overall Rating", "Potential", "Club", etc.). Esta documentación es suficiente para el proceso de transformación a RDF.

R6. Datos Provenientes de Múltiples Fuentes

GRADO DE SATISFACCIÓN: Parcial.

Discusión: El archivo CSV es un único conjunto de datos. No obstante, nuestro proyecto se centrará en la tarea de "Linking" con datos externos (como datos geográficos de los clubes de GeoNames o datos biográficos de DBPedia), por lo que el grafo RDF final sí integrará múltiples fuentes de información, satisfaciendo el espíritu del requisito.
