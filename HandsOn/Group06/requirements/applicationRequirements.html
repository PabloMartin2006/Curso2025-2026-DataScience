Documento entregable: applicationRequirements.html

1. Resumen de la aplicación

Nombre provisional: Mapa de Hospitales y Servicios

Objetivo: Crear una aplicación web que permita a ciudadanos y personal municipal buscar y explorar hospitales y centros de salud dentro de la ciudad, ver información clave y obtener filtros y listados. Los estudiantes transformarán los CSV seleccionados a RDF, enlazándolos con entidades geográficas (Wikidata / GeoNames) y publicarán un endpoint SPARQL.

3. Requisitos funcionales

  1. RF1 — Búsqueda por texto: Buscar por nombre y dirección.
  2. RF2 — Filtro por atributos: Filtrar por tipo de centro, servicios, distrito, etc.
  3. RF3 — Mapa interactivo: Visualización en mapa con popups de detalle.
  4. RF4 — Listado y detalles: Listado paginado y página de detalle.
  5. RF5 — Exportar resultado: Exportar resultados a CSV.
  6. RF6 — Endpoint SPARQL: Consumir datos RDF publicados por el grupo vía SPARQL.

4. Requisitos no funcionales

  • RNF1 — Accesibilidad básica.
  • RNF2 — Rendimiento: manejar hasta ~10k registros con paginado.
  • RNF3 — Portabilidad: Frontend desplegable en GitHub Pages.
  • RNF4 — Seguridad: no almacenar credenciales en el repositorio.

5. Roles de usuario

  • Ciudadano
  • Investigador / Estudiante
  • Administrador (opcional)

6. Casos de uso principales

  1. CU1: Buscar un hospital y ver su ubicación y teléfono.
  2. CU2: Filtrar por servicio y exportar la lista.
  3. CU3: Ejecutar consultas SPARQL sobre los datos enlazados.

9. Tecnologías sugeridas

  • Frontend: HTML5, CSS, JavaScript + Leaflet.
  • Transformación a RDF: OpenRefine / rdflib.
  • SPARQL endpoint: Apache Jena Fuseki.
  • Despliegue: GitHub Pages + contenedor para Fuseki.

10. Criterios de aceptación

  • Mostrar correctamente los puntos en el mapa usando coordenadas o geocodificando direcciones.
  • Datos RDF accesibles vía SPARQL con URIs enlazables.
  • Búsqueda y filtros mínimos implementados.
