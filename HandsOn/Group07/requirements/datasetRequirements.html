Dataset Requirements Evaluation

Dataset: Air Quality in Madrid (Daily Data – 2024)

This document reviews whether the selected dataset — “Calidad del aire. Datos diarios desde 2001 (2024)” — meets the requirements stated at the first part of this Hands-On assignment. The dataset was obtained from the Madrid City Council Open Data Portal and is available as a CSV file which contains daily air quality measurements collected across Madrid’s monitoring stations and it is regularly updated every month or so.

The evaluation considers six requirements in total: four mandatory (R1–R4) and two optional (R5–R6). Each one is discussed below, indicating whether it is satisfied, partially satisfied, or not satisfied.

Requirements Analysis

  • R1. Belonging to a Smart City domain:
    The dataset fits within the Environmental and Public Health domain, which is an essential component of Smart Cities. It includes indicators such as NO₂, PM₁₀, O₃, SO₂, and CO, all of which reflect Madrid’s environmental conditions. ✔ Satisfied
  • R2. Data available in CSV format:
    The dataset comes as a straightforward CSV file, no extra formatting or weird encodings, which made it easy to open and inspect. We used the file named air_quality_madrid_2024.csv stored in the csv/ directory. This makes it ready for later conversion into RDF without much preprocessing. ✔ Satisfied
  • R3. Open license:
    The dataset uses Madrid City Council’s Open Data License, which basically means anyone can reuse it as long as they give credit. This is perfect for this project as we can combine it with other sources without worrying. ✔ Satisfied
  • R4. Linkability with real-world entities:
    Each record includes station identifiers (ESTACION and PUNTO_MUESTREO) that can be linked to actual locations using coordinates from complementary datasets such as Calidad del aire – Estaciones. This makes it possible to connect the data with external sources like Wikidata or OpenStreetMap. ✔ Satisfied
  • R5. Dataset documentation (optional):
    The official documentation — “Interpreter of hourly, daily, and real-time air quality data files” — provides an explanation of the columns, pollutant codes (MAGNITUD), units of measurement, and validation flags. This made it easier to understand the data and its context. ✔ Satisfied
  • R6. Data from multiple sources (optional):
    The dataset currently originates from a single municipal source. However, it could be updated in the future by combining it with meteorological or traffic datasets to provide more context. ⚪ Partially satisfied

Conclusion

Overall, the Air Quality in Madrid (2024) dataset fulfills all four mandatory requirements (R1–R4) and meets some of the optional ones (R5–R6). It is therefore a good candidate for Linked Data transformation and integration, mainly for projects focused on urban sustainability and environmental monitoring.

In future work, it would be better to incorporate additional data sources. For instance, weather or traffic data in order to better understand the factors that influence air quality in Madrid.
