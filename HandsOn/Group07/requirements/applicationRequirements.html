Application Requirements

Application: Madrid Air Quality Dashboard

This document shows the main requirements and initial design draft of the application that will be developed based on the Air Quality in Madrid (2024) dataset. The goal of the application is to provide an accessible and visual way to explore air quality data across Madrid, helping users understand pollution trends and identify the areas most affected by poor air quality.

1. Application Overview

The Madrid Air Quality Dashboard will be a web-based application that displays real-time and historical air quality measurements collected from stations that monitor air quality around the city. The system will allow users to visualize pollution data through maps, charts, and filters by pollutant, date, and location.

2. Functional Requirements

  • FR1. Display air quality data by pollutant type (e.g., NO₂, PM₁₀, O₃, SO₂, CO).
  • FR2. Filter data by station, date range, and pollutant.
  • FR3. Provide an interactive city map showing the stations with color-coded pollution levels.
  • FR4. Allow users to visualize historical trends using charts and graphs.
  • FR5. Offer short explanations of each pollutant and its health effects if exposed for a long time.
  • FR6. Export selected data as CSV or JSON for further analysis.

3. Non-Functional Requirements

  • NFR1. The interface should be available on both desktop and mobile devices.
  • NFR2. Data visualizations must update dynamically and load within seconds.
  • NFR3. The system should be designed for easy integration with Linked Data sources (RDF).
  • NFR4. The design should follow accessibility standards, to ensure understanding for all users.
  • NFR5. All data used in the application must satisfy open data requirements.

4. User Interface Mock-ups

Main Dashboard:
- A top navigation bar with filters for pollutant, date range, and station.
- A map of Madrid in the center, with colored circles indicating air quality levels per station.
- Being over a station will display current pollutant values and the air quality index.
Trends & Analysis View:
- Line chart showing pollutant concentration changes over time.
- Bar graph comparing pollutants on a selected date.
- “Download data” button below the graphs.
Information Panel:
- Short explanations of each pollutant (e.g., “NO₂ – produced mainly by traffic emissions”).
- Simple recommendations, such as “Avoid outdoor exercise when PM₁₀ levels are high.”

5. Expected Impact

The application objective is to raise awareness about air quality conditions in Madrid and help both citizens and local authorities make better decisions. By visualizing pollution data in an intuitive way, it will make environmental information more transparent and accessible to the public.
